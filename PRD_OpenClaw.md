# OpenClaw PRD

## 1. 产品概述

### 1.1 产品名称
OpenClaw 公众号写作助手

### 1.2 产品定位
一个面向微信公众号创作者的桌面写作工具，帮助用户快速生成公众号文章初稿，并将生成内容直接发送到公众号草稿箱，缩短“选题到发稿”的操作链路。

### 1.3 产品目标
- 降低公众号文章初稿生成成本
- 支持原创写作与参考改写两种核心创作模式
- 支持多公众号账号配置与草稿箱投递
- 支持封面图上传并自动回填 `thumb_media_id`
- 将常用提示词、公众号账号、文章参数持久化，减少重复输入

### 1.4 目标用户
- 个人公众号作者
- 新媒体运营
- 内容工作室
- 需要批量试题、改写、发草稿的内容团队

## 2. 使用场景

### 2.1 原创写作
用户输入文章主题、目标读者、风格偏好、写作模式与长度，生成一篇可编辑的 Markdown 文章初稿。

### 2.2 参考改写
用户粘贴参考文章，指定改写目标、借鉴维度、参考程度和表达处理方式，生成一篇新的公众号文章。

### 2.3 草稿发送
用户将生成结果直接投递到微信公众号后台草稿箱，进入公众号后台继续润色、排版和发布。

### 2.4 多账号运营
用户维护多个公众号账号，在不同账号间切换并分别发送草稿。

### 2.5 封面图上传
用户选择封面图，通过后端上传到微信素材库并自动获得 `thumb_media_id`。

## 3. 产品形态

### 3.1 桌面端
- 旧版：PySide6 桌面客户端
- 新版：Tauri + React 桌面客户端

### 3.2 后端服务
- FastAPI 服务

### 3.3 CLI 工具
- 支持命令行直接生成文章 Markdown

## 4. 核心用户流程

### 4.1 原创生成流程
1. 打开桌面端
2. 选择或编辑提示词模板
3. 选择创作方式为“原创生成”
4. 输入主题、目标读者、风格偏好、长度、写作模式、表达处理
5. 点击“生成文章”
6. 查看并编辑生成结果
7. 复制内容或发送到公众号草稿箱

### 4.2 改写流程
1. 打开桌面端
2. 选择创作方式为“参考改写”
3. 粘贴参考文章
4. 配置改写目标、借鉴维度、参考程度、表达处理
5. 可选填写新主题、读者、风格偏好
6. 点击“生成文章”
7. 查看并编辑生成结果
8. 发送到公众号草稿箱

### 4.3 发草稿流程
1. 选择公众号账号
2. 确认 AppID、Secret、thumb_media_id
3. 填写或自动生成草稿标题
4. 点击“发送到公众号草稿”
5. 后端将 Markdown 转 HTML 并调用微信草稿接口
6. 返回 `media_id`，提示发送成功

## 5. 功能清单

## 5.1 桌面端功能

### 5.1.1 账号与服务配置
- 固定后端服务地址
- 启动后自动调用 `/health` 检测服务状态
- 显示当前运行平台信息

### 5.1.2 多公众号账号管理
- 新增公众号账号
- 删除公众号账号
- 切换当前生效账号
- 编辑账号名称
- 编辑公众号 AppID
- 编辑公众号 Secret
- 编辑封面图 `thumb_media_id`
- 本地持久化多个账号配置

### 5.1.3 提示词槽位管理
- 预置多个提示词槽位
- 槽位默认包含：
  - 系统提示词
  - 新媒体运营
  - 心理医生
  - 树洞学姐
- 切换槽位
- 修改槽位名称
- 修改槽位提示词内容
- 将当前槽位恢复为默认提示词
- 本地持久化槽位数据与当前激活槽位

### 5.1.4 文章创作参数
- 创作方式
  - 原创生成
  - 参考改写
- 文章长度
  - short
  - medium
  - long
- 写作模式
  - standard
  - story
  - case_study
  - listicle
  - analysis
- 表达处理
  - standard
  - conversational
  - de_ai
  - opinionated
- 主题输入
- 目标读者输入
- 风格偏好输入

### 5.1.5 改写专属参数
- 参考文章输入
- 改写目标
  - new_article
  - new_angle
  - more_conversational
  - more_actionable
- 借鉴维度
  - mixed
  - structure
  - tone
  - opening
- 参考程度
  - low
  - medium
  - high

### 5.1.6 生成结果区
- 展示 Markdown 生成结果
- 用户可在结果区手动编辑
- 自动从 Markdown 一级标题提取候选标题
- 用户可手动填写或修改草稿标题
- 可填写作者
- 可填写摘要

### 5.1.7 结果操作
- 生成文章
- 复制文章到剪贴板
- 清空生成结果
- 发送到公众号草稿箱

### 5.1.8 封面图上传
- 选择本地 JPG/PNG 图片
- 调用后端上传接口
- 上传成功后自动回填 `thumb_media_id`

### 5.1.9 本地持久化
- 当前公众号账号列表
- 当前激活账号
- 提示词槽位内容
- 当前激活提示词槽位
- 当前文章参数
- 最近生成结果
- 草稿标题/作者/摘要

## 5.2 后端服务功能

### 5.2.1 健康检查
- `GET /health`
- 返回服务可用状态

### 5.2.2 文章生成
- `POST /article/generate`
- 接收主题、长度、模式、系统提示词、改写参数等
- 支持原创生成
- 支持参考改写
- 调用 OpenAI 兼容模型接口生成 Markdown
- 返回文章正文与模型元信息

### 5.2.3 封面图上传
- `POST /wechat/upload_thumb`
- 上传图片文件到微信素材接口
- 返回 `thumb_media_id`
- 可选接收前端传入的 AppID/Secret

### 5.2.4 公众号草稿发送
- `POST /wechat/draft`
- 接收标题、Markdown 正文、作者、摘要、公众号配置
- 将 Markdown 转换为 HTML
- 为 HTML 自动加公众号阅读样式
- 调用微信草稿接口
- 返回 `media_id`

### 5.2.5 微信 Access Token 缓存
- 基于 AppID + Secret 进行 token 缓存
- 在过期前自动复用

## 5.3 CLI 功能

### 5.3.1 文章生成命令
- 命令入口：`python -m openclaw article`
- 支持参数：
  - `--topic`
  - `--audience`
  - `--style`
  - `--mode`
  - `--length`
  - `--output`
- 输出 Markdown 文件

## 6. 详细功能要求

## 6.1 文章生成要求
- 原创模式下，主题必填
- 改写模式下，参考文章必填
- 改写模式下，参考文章长度过短时应提示用户补充
- 生成期间按钮需禁用，避免重复提交
- 生成失败时应提示错误信息
- 生成结果默认以 Markdown 格式输出

## 6.2 草稿发送要求
- 没有生成结果时不可发送
- 草稿标题为空时，优先自动从 Markdown 一级标题中提取
- 如果 Markdown 中无一级标题，则退回使用主题
- 发草稿前应读取当前生效公众号账号配置
- 当 `thumb_media_id` 缺失时，应明确提示用户

## 6.3 图片上传要求
- 支持 PNG / JPG / JPEG
- 上传成功后自动回填 `thumb_media_id`
- 上传失败时提示具体错误

## 6.4 多账号要求
- 至少保留一个公众号账号
- 删除账号后应自动切换到有效账号
- 所有账号配置需本地持久化

## 6.5 提示词槽位要求
- 槽位支持切换
- 修改后自动保存
- 恢复默认后立即替换当前内容

## 7. 非功能要求

### 7.1 易用性
- 页面结构面向普通内容用户，而不是程序员
- 尽量减少技术配置暴露
- 核心操作链路应控制在一屏主工作流内

### 7.2 性能
- 生成请求应异步处理
- 长时间请求期间提供状态提示
- 切换账号、切换提示词不应出现明显卡顿

### 7.3 稳定性
- 后端接口异常时给出清晰提示
- 网络错误、微信接口错误、模型错误应分开提示

### 7.4 数据安全
- 公众号账号配置保存在本地
- API Key 不应写入远端仓库
- 环境变量仅用于本地或服务端部署
